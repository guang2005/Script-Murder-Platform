# Script-Murder-Platform
剧本杀管理平台 / 剧本杀系统

## 📖 项目介绍
这是一个基于 SpringBoot 开发的剧本杀管理平台，支持用户登录签到,发布剧本博客,博客点赞,关注博主,优惠场次秒杀,只能客服等功能。
适用于线下门店、线上预约系统使用。

## 🛠️ 技术栈
- 后端：SpringBoot、SpringMVC、MyBatis/MyBatis-Plus
- 数据库：MySQL
- 开发工具：IDEA、Maven、Git
- 前端：Vue / 原生页面（根据你实际情况改）

- ## 🎯 功能模块
- 用户模块
- 商铺模块
- 优惠卷秒杀模块
- 博客模块
- ai模块

## 🚀 如何运行
1. 导入项目到 IDEA
2. 创建数据库，执行 SQL 文件
3. 在虚拟机启动Redis
4. 在本机部署RocketMQ(建议docker部署)
5. 本机部署ollama,配置模型
6. 修改 application.yml 数据库配置
8. 启动项目，访问 localhost:端口

   ## 项目说明
本项目基于开源项目「黑马点评」进行二次开发，在原有基础上做了以下核心改进与优化：

### 📌 技术选型优化
- **消息队列**：将原方案替换为 **RocketMQ**。
  作为初学者，我选择本机部署以快速上手；**生产环境推荐使用 Docker 部署**，后续我也会逐步迁移至 Docker 方案。
- **智能客服功能**：新增 AI 智能客服模块，基于 Ollama 生态实现。
  - 大语言模型：`qwen2.5:7b-instruct`（轻量易用，适合入门）
  - 向量嵌入模型：`nomic-embed-text`（部署简单，文本向移成本低）
  > 推荐初学者直接使用这两个模型，学习成本低、开箱即用。

### ⚡ 缓存方案重构
针对缓存问题，我重新设计了更优雅的解决方案：
- **缓存穿透**：使用 **布隆过滤器** 拦截不存在的 key，避免无效请求穿透到数据库。
- **缓存击穿**：采用 **互斥锁 + 逻辑过期** 方案，在保证数据一致性的同时，避免热点 key 失效时大量请求压垮数据库。
- **缓存雪崩**：通过 **多级缓存架构**（本地缓存 + 分布式缓存）分散风险，配合随机过期时间防止缓存同时失效。

> 最初版本我尝试过「空值 + 互斥锁 + 逻辑过期」的组合，但代码冗余且数据安全保障不足，最终选择了这套更简洁、健壮的方案。
