# Script-Murder-Platform
剧本杀管理平台 / 剧本杀系统

## 📖 项目介绍
这是一个基于 SpringBoot 开发的剧本杀管理平台，支持用户登录签到,发布剧本博客,博客点赞,关注博主,优惠场次秒杀,只能客服等功能。
适用于线下门店、线上预约系统使用。

## 🛠️ 技术栈
- 后端：SpringBoot、SpringMVC、MyBatis/MyBatis-Plus
- 数据库：MySQL
- 开发工具：IDEA、Maven、Git
- 前端：Vue / 原生页面（根据你实际情况改）

- ## 🎯 功能模块
- 用户模块
- 商铺模块
- 优惠卷秒杀模块
- 博客模块
- ai模块

## 🚀 如何运行
1. 导入项目到 IDEA
2. 创建数据库，执行 SQL 文件
3. 在虚拟机启动Redis
4. 在本机部署RocketMQ(建议docker部署)
5. 本机部署ollama,配置模型
6. 修改 application.yml 数据库配置
8. 启动项目，访问 localhost:端口

   本项目是基于网络上的黑马点评二次开发,接下来我讲一下自己的主要改动,消息队列采用了RocketMQ,因为我也是初学者目前觉得本机部署方便,但是建议大家doker部署后续我也会换docker部署,新增了智能客服功能
   采用的ollama里的qwen2.5:7b-instruct和nomic-embed-text,因为这两个简单而且方便,建议初学者就采用这两个模型,使用布隆过滤器解决缓存穿透,互斥锁+逻辑过期解决缓存击穿,多级缓存解决缓存雪崩,这里其
   实有很多技术选型,我第一版是选择空值+互斥锁和逻辑过期,但是我觉得代码过于臃肿,而且并不能很好的保证数据安全,还是有点瑕疵,所以最后选择了这版方案,很多地方写的不好,但希望能给和我相同的初学者一些帮助
